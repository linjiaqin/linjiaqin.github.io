资源参数：
（1）connect_time 一个会话能连接到数据库的总时间（以秒计）
（2）session_per_user 用户能打开的会话的最大数目

(1)failed_login_attempts 设置一个用户在被锁定前可尝试的登录次数

用户管理

创建一个新的数据库时，最开始的用户是应用或模式的拥有者，然后才穿件日常使用的最终用户
应用的拥有者关心的是分配空间和授权以便在数据库中创建对象。
最终用户，关心的是他们对各种对象的访问权限和资源限制。

oracle 有几个基本用户：
(1)sys 是超级用户 登录的命令为 sqlplus sys as sysdba
(2)system

所有的用户都需要临时表空间和默认表空间
在oracle中，可在数据库创建的过程中为所有的用户创建一个临时表空间和默认表空间
所以创建数据库时用户不一定必须制定。

细粒度数据访问

定义：可以允许所有用户访问一个表，但是限制各个用户只能访问表中与他们有关的那些行。这样的限制对用户来说是透明的。

oracle使用两种机制来实施数据库的细粒度的安全：应用环境和细粒度访问控制策略。

oracle使用VPD虚拟专用数据库来实现

通过VPD，每个用户只看到表数据的一部分。这种行级的安全是通过直接附加一个安全策略到数据库对象实施的。
原理：VPD使用某种查询重写来限制用户访问表和视图的行。安全策略附加到希望控制访问的表，并且编写存储过程来修改针对这些表构造的sql语句。

细粒度访问控制：

细粒度访问控制允许限制用户，使他们只能使用你让他们访问和修改的数据。使用策略函数附加到想保护的表或者视图上，它使用动态可修改的语句将用户约束或限制在表、视图或者同义词的某些成分中。在分析用户的sql语句时，细粒度访问控制让oracle自动估计策略函数，oracle将在动态修改查询后执行它。

需要的条件：
• 设置应用环境的程序包
• 创建应用环境
• 设置应用环境
• 创建一组安全策略
步骤：
1) 给用户的DML语句动态增加一个谓词的策略函数。谓词是基于某个操作符的where子句。比如给orders表
2) 用户输入下面的一条语句select * from orders。
3) Oracle将使用你创建的策略函数动态修改用户的语句。
4) Oracle使用SYS_CONTEXT(‘USERENV’,’SESSION_USER’)返回的用户名对原查询进行修改，从而限制从oracle表返回的数据仅为该用户的数据。
